# Claude Code 接入 stage-bridge 的 System Prompt 模板

将以下内容追加到 Claude Code 项目的 `CLAUDE.md`（位于仓库根目录）。

---

## 多 Agent 平台协作协议

本仓库（`${SERVICE_NAME}`）受 stage-bridge 多 Agent 开发平台调度。在响应平台分配的研发任务时，你必须严格遵循下面的协作协议；在用户直接交互的开发场景下（无 `REQ_ID` 环境变量），可忽略本协议。

### 启动协议

收到任务后立即在 Bash 工具中执行：

```bash
python $STAGE_BRIDGE_DIR/scripts/register_agent.py \
  --capabilities backend --service ${SERVICE_NAME} --repo-path "$(pwd)"

nohup python $STAGE_BRIDGE_DIR/scripts/heartbeat.py --loop 10 > /tmp/hb.log 2>&1 &
```

### 任务执行协议

```bash
# 1. 抢占任务（exit 1 = 已被抢占，直接结束）
python $STAGE_BRIDGE_DIR/scripts/claim_task.py "$REQ_ID" "$TASK_NAME"

# 2. 加载上下文
python $STAGE_BRIDGE_DIR/scripts/read_context.py "$REQ_ID"
```

3. **真正的编码工作**：使用 Read / Edit / Write / Bash 工具在 `$REPO_PATH` 下完成需求实现。每完成一个用户可感知的步骤（创建分支、实现一个接口、通过单元测试），调用一次 `log_step.py`。

```bash
python $STAGE_BRIDGE_DIR/scripts/log_step.py "$REQ_ID" "<message>"
```

4. **创建 PR 并标记任务**：

```bash
gh pr create --title "[${REQ_ID}] ${TASK_NAME}" --body "<auto-generated description>"
PR_URL=$(gh pr view --json url -q .url)

python $STAGE_BRIDGE_DIR/scripts/write_artifact.py "$REQ_ID" pr_url "$PR_URL"

python $STAGE_BRIDGE_DIR/scripts/complete_task.py "$REQ_ID" "$TASK_NAME" \
  --await-review --pr-url "$PR_URL" \
  --meta "{\"branch\":\"$(git branch --show-current)\",\"commit\":\"$(git rev-parse HEAD)\"}"
```

### 失败协议

任何不可恢复的错误（如依赖冲突、需求不清、测试无法通过）都必须显式上报失败：

```bash
python $STAGE_BRIDGE_DIR/scripts/fail_task.py "$REQ_ID" "$TASK_NAME" \
  --error "<concise reason>" --retry-hint manual
```

`retry-hint` 取值：`retry`（可自动重试）/ `blocked`（依赖未满足）/ `manual`（需人工干预）。

### 反馈修复协议（仅当被告知"修复模式"时）

```bash
python $STAGE_BRIDGE_DIR/scripts/feedback_listen.py "$REQ_ID" "${SERVICE_NAME}" --timeout 600
```

输出的 JSON 包含 `payload.error_summary`、`payload.failed_cases`、`payload.log_url` 等字段，据此诊断、修复、提交，然后：

```bash
python $STAGE_BRIDGE_DIR/scripts/feedback_resolve.py "$REQ_ID" "${SERVICE_NAME}" \
  --summary "<what was fixed>" --commit "$(git rev-parse HEAD)"
```

### 硬性约束

绝不在未抢占任务前开始编码。绝不直接 push 到 `main`，必须使用 `feature/${REQ_ID}-<desc>` 分支。
绝不跳过 `complete_task.py` 或 `fail_task.py`，否则会被 Watchdog 判定为僵尸任务并强制重试。
绝不绕过 stage-bridge 脚本直接 `curl` Consul，路径约定可能在框架层调整。
